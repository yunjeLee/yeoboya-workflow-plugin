# Prompt Refiner

모든 skill은 시작 시 이 지침을 따라 사용자 입력을 정제한다.

## 동작 흐름

사용자 입력을 수신하면 아래 순서로 처리한다.

1. **변환 단계**: 모호한 표현을 구조화된 요구사항으로 정제한다.
2. **불명확 감지 단계**: 여전히 모호한 부분이 있는지 판단한다.
   - 없음 → 정제된 요구사항을 출력하고 스킬을 진행한다.
   - 있음 → 불명확한 부분을 1개씩 질문한다. 답변 반영 후 재감지한다.

## 핵심 규칙

- 질문은 반드시 1개씩만 한다.
- 판단이 불확실하면 가정하고 "가정한 사항"에 명시한다.
- 사용자가 이미 상세하게 입력했으면 변환만 하고 질문을 생략한다.

## 정제 출력 형식

정제 완료 후 반드시 아래 형식으로 출력한다.

```
[요구사항 정제 완료]
- 작업 유형: [기능 개발 / 버그 수정 / 마이그레이션 / 신규 앱]
- 핵심 목표: [한 줄 요약]
- 상세 요구사항:
  1. ...
  2. ...
- 가정한 사항: [사용자가 명시하지 않았지만 가정한 것. 없으면 "없음"]
- 확인된 제약: [플랫폼, 기술 스택 등. 없으면 "없음"]
```
