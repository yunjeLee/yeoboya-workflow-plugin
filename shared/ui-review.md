# UI Review

UI 작업이 감지될 때만 이 지침을 따른다.

## UI 작업 감지 기준

각 구현 Task 시작 전 아래 기준으로 UI 작업 여부를 판단한다.

- Composable / View / Fragment / Layout 파일 수정
- 색상 / 여백 / 애니메이션 관련 코드
- 화면 단위 작업 (Screen, Activity, ViewController)

판단 결과:
- **UI 작업 아님** → 기존 로직 플로우 그대로 진행한다.
- **UI 작업 맞음** → 아래 UI 검토 체크포인트를 실행한다.

## UI 검토 체크포인트

### 1. @Preview 포함 코드 생성

Composable 코드 작성 시 반드시 @Preview를 포함한다.

```kotlin
@Preview(showBackground = true)
@Composable
fun ComponentNamePreview() {
    AppTheme {
        ComponentName(
            // 대표적인 상태값으로 미리보기 구성
        )
    }
}
```

### 2. 피드백 루프 (최대 10회)

코드 생성 후 아래 메시지를 출력하고 사용자 응답을 기다린다.

```
─────────────────────────────────────────────
[UI 검토 요청] (N/10회)
Android Studio에서 Preview를 확인하세요.

- 스크린샷을 첨부하면 분석 후 수정합니다.
- 0을 입력하면 중단하고 사용자에게 인계합니다.
- y를 입력하면 완료로 처리하고 다음 단계로 진행합니다.
─────────────────────────────────────────────
```

응답에 따라 처리:
- **스크린샷 첨부**: 문제를 분석하고 수정한 뒤 루프를 반복한다. 회차를 1 증가시킨다.
- **0 입력**: 현재 상태를 저장하고 아래 인계 메시지를 출력한 뒤 루프를 종료한다.
- **y 입력**: 완료로 처리하고 다음 Task로 진행한다.
- **10회 도달**: 자동으로 0과 동일하게 처리한다.

### 3. 인계 메시지 형식 (0 입력 또는 10회 도달 시)

```
[UI 인계]
자동 수정을 종료합니다.
현재까지의 코드를 저장했습니다.
직접 수정 후 '/ui-preview-loop [컴포넌트명]'으로 재시작하거나
다음 Task 진행을 원하면 'continue'를 입력하세요.
```
