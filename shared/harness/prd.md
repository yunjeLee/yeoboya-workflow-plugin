# prd 모듈

호출자 (`/harness`, `/harness-edit`) 가 Read tool 로 읽고 지침을 따른다. 5 개의 대화형 질문을 1 개씩 순차로 받는다.

## 대상 파일

`docs/PRD.md`

## 사전 스캔

질문 시작 전 아래 소스를 Read / Glob / Bash 로 확인한다. 결과는 질문별 예시 생성에 사용한다.

| 소스 | 용도 |
|------|------|
| `README.md` 상단 1~2 문장 | 앱 설명, 외부 문서 링크 |
| `settings.gradle.kts` / `Package.swift` | 모듈 / 타겟명 (앱 영역 추론) |
| 패키지 루트 디렉토리 이름 | 도메인 추론 (`booking`, `payment`, `chat` 등) |
| `git tag --list` + `git log --oneline | wc -l` | 출시 단계 추정 |

### 결과 형식

- 앱 한 줄 요약 (README + 패키지명)
- 사용자 / 사용 맥락 키워드 추론
- 출시 단계 추정 (MVP / Growth / Mature)
- 외부 문서 링크 후보

## 섹션 목록

| 섹션 ID | 헤더 | 타입 | 질문 수 |
|--------|-----|-----|--------|
| s1 | `## 이 앱이 하는 일` | 대화형 | 1 |
| s2 | `## 주요 사용자 & 사용 맥락` | 대화형 | 1 |
| s3 | `## 핵심 가치` | 대화형 | 1 |
| s4 | `## 출시 상태` | 대화형 | 1 |
| s5 | `## 외부 상세 문서` | 대화형 | 1 |

> `## 안 하는 것` 섹션은 [TBD] 고정 (질문 없음). 팀원이 수동으로 채우게 둔다.

## 섹션별 생성 로직

각 질문은 아래 공통 출력 형식을 따른다.

### 공통 출력 형식

```
[질문 N] {질문 내용}

레포 스캔 결과 (추정):
  {해당 질문에 쓴 스캔 소스 요약}

아래 중 가깝거나 직접 입력해주세요:
  1) {예시 1} (매칭: {근거})
  2) {예시 2} (매칭: {근거})
  3) {예시 3} (매칭: {근거})
  4) 직접 입력

번호 선택 또는 직접 입력 (건너뛰기 시 Enter → [TBD]):
```

### 예시 생성 소스 (질문별)

| 섹션 ID | 질문 | 예시 생성 소스 |
|--------|------|------------|
| s1 | 이 앱이 뭘 하는지 1~2 문장 | 패키지명, 모듈명, README 한 줄 |
| s2 | 주 사용자 및 사용 맥락 | README 키워드, 모듈명 (`feature:reservation` → "호텔 예약 고객" 등) |
| s3 | 핵심 가치 1~3 개 | 일반 템플릿 ("유지보수성 > 화려한 코드", "빠른 학습 곡선", "실사용자 피드백 우선") |
| s4 | 출시 단계 (MVP / Growth / Mature) | `git tag` + commit 수로 추정 (tag 없음 + 커밋 < 50 → MVP, tag 있음 + 활발 → Growth, tag 있음 + 활발 X → Mature) |
| s5 | 외부 상세 문서 링크 | README 내 http(s) 링크 탐색 |

스캔 소스 부재 시 (레포가 비어있거나 README 없음): 일반 템플릿 예시 2~3 개 제시, 매칭 근거 위치에 `(일반 템플릿)` 표기.

### 건너뛰기 처리

빈 줄 / `건너뛰기` / `없음` / `스킵` / `skip` / `-` → `[TBD]`.

## 출력 템플릿

````markdown
# 프로덕트 개요

## 이 앱이 하는 일
{s1 답변 또는 [TBD]}

## 주요 사용자 & 사용 맥락
{s2 답변 또는 [TBD]}

## 핵심 가치
{s3 답변 — 리스트 형태로 렌더링. 없으면 [TBD]}

## 출시 상태
- 플랫폼: {자동 감지 결과}
- 단계: {s4 답변 또는 [TBD]}

## 안 하는 것 (out of scope)
- [TBD]

## 외부 상세 문서
{s5 답변 또는 [TBD]}
````
