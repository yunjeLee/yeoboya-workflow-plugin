# Onboard — Case 3 브리핑 모드

`/onboard` 의 Case 3 (핵심 문서 6 종 전부 존재) 진입 시 이 문서의 지침을 따른다.

## 목적

신규 팀원에게 프로젝트 구조와 업무 워크플로우를 한눈에 보여준다. **파일은 일체 수정하지 않는다.**

## 절차

### Step 1: 필수 파일 Read

- `CLAUDE.md`
- `docs/PRD.md`
- `docs/ARCHITECTURE.md`
- `docs/ADR.md`
- `docs/TESTING.md`
- `docs/CONVENTIONS.md`
- `docs/UI_GUIDE.md` (있으면)
- `docs/WORKFLOW.md`

추가 정보 수집:
- `git remote get-url origin` 으로 레포 URL 조회 (실패 시 "레포 URL 없음 (로컬 전용)" 으로 표기)
- 빌드 시스템 감지: `build.gradle.kts` 또는 `build.gradle` 존재 시 Android, `*.xcworkspace` 또는 `*.xcodeproj` 존재 시 iOS (Glob 으로 탐색)
  - Android: `build.gradle.kts` Read → 주요 task 확인
  - iOS: `*.xcodeproj/project.pbxproj` Read → scheme 이름 추출

### Step 2: 정적 브리핑 출력

아래 포맷으로 출력한다. 각 섹션 오른쪽의 `(docs/XXX.md)` 출처 표기는 **반드시 포함** — 신규 팀원이 직접 파일을 열어볼 때 경로 파악용.

### Step 3: 동적 후속 대응

사용자 입력을 파싱해 아래 매핑에 따라 응답:

| 사용자 입력 | 동작 |
|------------|------|
| `1 {모듈명}` | `Glob` 으로 `{모듈명}/src/main/kotlin/**/*.kt` 또는 `Sources/{모듈명}/**/*.swift` 나열 → 진입점(Activity/Application/@main) 파일 Read → 핵심 파일 설명 |
| `2` | `docs/ADR.md` 전문 출력 + 아래 두 커맨드 결과 병합 제시: <br>① `Bash: git log --all --since='30 days ago' --grep='ADR' --oneline` (커밋 메시지에 ADR 태그 단 경우) <br>② `Bash: git log -- docs/ADR.md --since='30 days ago' --oneline` (ADR.md 파일 변경 이력 — 팀이 ADR 태그를 안 써도 동작) <br>둘 다 빈 결과면 "최근 30 일 내 ADR 변경 없음" 표기 |
| `3 {기능명}` | `feature/{기능명}` 또는 `features/{기능명}` 모듈 Glob → 관련 UseCase / Repository 파일 Read → 데이터 흐름 설명 |
| `4` | `build.gradle(.kts)` 의 주요 task + `Glob: .github/workflows/*.yml` → 각 워크플로우 목적 요약 |
| `5 {키워드}` | `Glob: **/*{키워드}*.{kt,java,swift}` → 상위 10 개 파일 + 각 1 줄 설명 |
| `6 {스킬명}` | `skills/{스킬명}/SKILL.md` Read → 스킬 목적 요약 + 실전 시나리오 1 개 생성 |
| 자유 텍스트 | 입력 분석 후 관련 파일 Read → 답변 |

**동적 후속은 1 회 응답 후 "추가로 궁금한 게 있나요?" 로 재질문하며 대화를 유지한다.** 사용자가 "종료" 또는 "없음" 이라고 답하면 종료.

## 유의사항

- 파일은 **절대 수정하지 않는다**. Case 3 는 read-only 모드다.
- 파일 일부가 `[TBD]` 로 채워진 경우, 해당 섹션에는 "`[TBD]` (아직 채워지지 않음)" 으로 표기하고 "`/onboard` 를 다시 실행하면 보완할 수 있습니다" 안내.
- 출력이 너무 길면 섹션 단위로 잘라 표시하고 `[Enter] 키로 계속` 같은 pager 형식은 사용하지 않는다 (Claude Code 터미널 환경 고려).
- CLAUDE.md 에서 특정 섹션(CRITICAL 규칙, 피해야 할 것 등)을 찾지 못할 경우 해당 섹션을 "[섹션명] 섹션 없음" 으로 표기한다.
