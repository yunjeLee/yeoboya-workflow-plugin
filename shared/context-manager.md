# Context Manager

모든 skill은 이 지침을 따라 컨텍스트를 관리한다.

## Phase / Task 계층

- **Phase**: 큰 작업 단계 (brainstorming → writing-plans → 실행 → done)
- **Task**: Phase 내 세부 작업 단위

예시 흐름 (/feature "사용자 인증"):
```
Phase: brainstorming → Phase: writing-plans → task-1 구현 → task-2 구현 → done
```

## 핵심 원칙: 1 세션 = 1 목적

각 세션은 하나의 Phase 또는 하나의 Task만 수행한다.
Phase/Task 완료 후 반드시 상태를 저장하고 /clear를 안내한다.

## 상태는 plan.md 에 산다

별도 상태 파일은 쓰지 않는다. `{skill}-plan.md` 의 Task 체크리스트(`- [ ]` / `- [x]`)가 유일한 완료 기록이다. /clear 후 재시작할 때 Claude 는 plan.md 를 Read 해 진행 상황을 복원한다.

Phase 재시작 메시지의 `이전 단계` / `다음 단계` / `관련 파일` 값은 plan.md 와 직전 대화에서 Claude 가 직접 추출한다.

## /clear 유도 시점 3가지

### ① Phase 전환 — **필수**

Phase 가 끝나면 **반드시** 아래 형식으로 재시작 메시지를 출력한다. **생략 불가.**

**발동 조건**: `{skill}-plan.md` 의 Phase 체크박스 중 하나가 `- [ ]` 에서 `- [x]` 로 변경된 직후.

```
─────────────────────────────────────
[skill명] 재개
- 이전 단계: [completedPhase]
- 다음 단계: [nextPhase] (예: "task-2: 구현 시작", "done: 모든 작업 완료")
- 관련 파일: [files 목록]
- 플랫폼: [platform]
─────────────────────────────────────
```

"/clear 후 위 메시지를 붙여넣어 재시작하세요."

### ② 컨텍스트 50% 초과 — 권장 (자동 측정 수단 검토 중)

> ⚠️ **상태**: 현재 Claude 가 자기 토큰 사용량을 직접 측정할 공식 수단이 없다. 이 섹션은 "권장" 으로만 작동하며, 자동 강제 장치는 없다. 추후 transcript 파일 크기 기반 Hook 또는 다른 수단이 확정되면 "필수" 로 전환될 예정.

Claude 가 자발적으로 체크한다. 50% 를 넘는다고 느끼면 아래를 **권장** 한다 (필수 아님):

```
컨텍스트 사용량이 50% 를 넘었을 가능성이 있습니다.
지금 상태를 저장하고 /clear 후 재시작하는 것을 권장합니다.
계속 진행하시겠습니까? (y/n)
```

### ③ 작업 이탈 감지 — **필수**

사용자 입력이 현재 진행 중인 skill 범위를 명백히 벗어나면 **반드시** 전환 알림을 출력한다. "판단" 이 아니라 **규칙 매칭** 으로 결정한다.

**발동 규칙** (AND 조건):

1. 현재 진행 중인 skill 확인 (워킹 디렉토리에 `{skill}-plan.md` 가 있고 미완료 Task 가 남아있음)
2. 사용자 입력에 아래 **전환 키워드** 포함:

   | 현재 skill | 전환 키워드 예시 (전환 대상) |
   |-----------|----------------------------|
   | `/feature` 진행 중 | `버그`, `bug`, `에러`, `error`, `크래시`, `crash`, `작동 안`, `안 됨`, `깨짐` → `/bugfix` |
   | `/bugfix` 진행 중 | `기능 추가`, `새 기능`, `feature`, `만들어줘`, `구현해줘`, `새 화면` → `/feature` |

3. **제외 규칙** (false positive 방지):
   - 키워드가 현재 plan.md 의 미완료 Task 명/설명에 이미 등장하면 → **알림 안 함** (같은 범위 내 세부 논의)
   - 사용자가 같은 feature 내에서 자잘한 수정을 요청하는 경우 → **알림 안 함**
   - 3 개 규칙 모두 충족 시에만 알림 출력

**출력 형식**:

```
현재 작업([현재 skill])과 관련 없는 요청으로 감지됐습니다.
비용 절약 및 context rot 방지를 위해 /clear 후 /[권장 skill]로 새로 시작하는 걸 권장합니다.

현재 세션에서 계속 진행하려면 'y' 를 입력하세요.
```

강제 전환이 아니라 **알림 출력** 이 필수다. 'y' 입력 시 현재 세션을 유지한다.
